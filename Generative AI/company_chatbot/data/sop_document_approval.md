# Standard Operating Procedure: Document Approval Workflow

**Document ID:** SOP-001  
**Version:** 2.1  
**Effective Date:** January 1, 2024  
**Last Review:** December 15, 2023  
**Department:** Quality Assurance  

## Purpose
This SOP defines the standard process for approving internal company documents including policies, procedures, work instructions, and forms.

## Scope
This procedure applies to all employees who create, review, or approve company documents.

## Definitions
- **Document Owner:** The individual responsible for creating and maintaining the document
- **Reviewer:** Subject matter expert who verifies technical accuracy
- **Approver:** Manager or executive with authority to approve document implementation
- **Version Control:** System for tracking document changes and revisions

## Procedure

### 1. Document Creation (Document Owner)
1.1. Create new document using company template  
1.2. Include all required sections: title, purpose, scope, effective date  
1.3. Assign unique document ID following naming convention: [DEPT]-[TYPE]-[NUMBER]  
1.4. Save draft in shared drive under "Draft Documents" folder  

### 2. Internal Review (Document Owner)
2.1. Conduct self-review for completeness and clarity  
2.2. Check spelling, grammar, and formatting  
2.3. Verify all references and citations are accurate  
2.4. Update version number to X.0 for new documents  

### 3. Peer Review (Reviewer)
3.1. Document owner submits to designated reviewer(s)  
3.2. Reviewer has 5 business days to complete review  
3.3. Reviewer checks for:
   - Technical accuracy
   - Compliance with regulations
   - Clarity and usability
   - Consistency with existing procedures  
3.4. Reviewer provides feedback through comment function  
3.5. Document owner addresses all comments and resubmits  

### 4. Management Approval (Approver)
4.1. Once peer review is complete, submit to department manager  
4.2. Manager reviews document within 3 business days  
4.3. Manager either:
   - Approves: Signs and dates document
   - Requests changes: Returns with specific feedback
   - Rejects: Provides rationale for rejection  

### 5. Final Approval (Quality Assurance)
5.1. All approved documents must receive final QA approval  
5.2. QA verifies:
   - All required sections present
   - Proper version control applied
   - Appropriate signatures obtained
   - Correct filing location identified  
5.3. QA stamps document as "Approved for Use"  

### 6. Publication and Distribution
6.1. Move approved document to "Active Documents" folder  
6.2. Update document register with new entry  
6.3. Notify affected employees via email  
6.4. Provide training if document requires new skills or knowledge  
6.5. Archive previous version (if applicable)  

### 7. Document Review Cycle
7.1. All documents must be reviewed annually  
7.2. Document owner receives reminder 30 days before review due date  
7.3. If no changes needed, update "Last Review" date and re-approve  
7.4. If changes needed, follow full approval workflow  

## Roles and Responsibilities

**Document Owner:**
- Create and maintain document
- Address reviewer feedback
- Ensure timely reviews and updates
- Archive obsolete versions

**Reviewer:**
- Provide timely, constructive feedback
- Verify technical accuracy
- Ensure regulatory compliance

**Approver:**
- Review and approve/reject documents
- Ensure alignment with company strategy
- Authorize implementation

**Quality Assurance:**
- Final verification of all documents
- Maintain document register
- Monitor compliance with this SOP
